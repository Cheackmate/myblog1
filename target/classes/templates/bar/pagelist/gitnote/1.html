关联github方法
1.验证秘钥
1、在黑屏终端输入： ssh-keygen -t rsa -C "自己的邮箱地址"，输入密码

2、记录相关.ssh 目录的位置

3、可以再本地电脑中找到.ssh目录：
id_rsa私有秘钥

id_rsa.pub 共有秘钥
把私有秘钥复制到自己github的秘钥上，然后记录

8、ssh -T git@github.com 测试秘钥是否通过，没有报错就没有通过，要是可能不会变绿，之后使用的时候就会变绿了。

2.上传和下载数据
1、把该文件夹变成仓库：git init

2、关联远程仓库，电脑和仓库关联

cd到项目文件夹下：git remote add origin 仓库地址
3、先拉取远程仓库文件：git pull origin master --allow-unrelated-histories

5、在仓库中创建文件，然后把该文件提交到相应的暂存区：

git add <文件名>

git add . 添加当前目录下的所有文件

6、把暂存区的文件提交到相应的仓库：

git commit -m "备注信息"

7、把本地的文件推到远程： git push origin master

3.创建分支并把分支的数据推送到master分支
1.首先切换到分支；

git checkout hellomonkey
2.使用git pull 把分支代码pull下来；

git pull
3.切换到主分支；

git checkout master
4.把分支的代码merge到主分支；

git merge hellomonkey
5.git push推上去ok完成,现在 你自己分支的代码就合并到主分支上了。

git push
总结：同样的道理，主分支的文件也可以合并的分支上

4.分支的创建和切换
创建分支： git branch 分支名

切换分支： git checkout 分支名

创建切换同时进行： git checkout -b 分支名

列出所有分支： git branch

把工作成果合并到master分支上(先切换到master)： git merge 分支名

删除分支： git branch -d 分支名

删除远程分支： git push origin:分支名
git push origin --delete 分支名

查看分支合并图：git log --graph

推送分支 ：
git push origin master
git push origin dev

抓取分支：git checkout -b dev origin/dev
